## Phase 1 (completed)

in this application, we already support a "cut and paste" feature where the user can cut and paste files and folders into a different folder. the way this works currently is that the user will use the checkboxes that appear in the `BrowseView.tsx`. the checkbox itself has a component that renders it which is named `SelectionCheckbox` which is actually embedded into our "Entry" components (entry components are all named like `src/components/entries/*Entry.tsx`, by the way). once the user has one or more selections made in these check boxes buttons named "Cut" (with flyover "Cut Selected Items") and "Del" (with flyover "Delete Selected Items") will begin to appear at the top of the `BrowseView.tsx` to allow the user to be able to cut or paste into the current folder that they're browsing. i'm telling you all this so you can understand what we currently have working already and the cut and paste feature works well. 

however, another way a user can paste some files or folders that have been cut using the "Cut" button, is by selecting the "Paste into Folder" menu item that appears in the right-mouse-click context menu of the `IndexTreeView.tsx` component, which appears whenever the user right clicks a folder in the IndexTreeView. 

so the feature we're going to add is that we're going to make this process a lot more streamlined where the user can drag a single file or folder from the `BrowseView.tsx` and drop it onto a folder item in the `IndexTreeView`, and that will of course move the file or folder on the file system into the target folder it was dropped on. so based on what I've described above most of the hard work for doing the file move operation already exists in our code, although it might need to be refactored slightly, to support our new Drag-and-drop feature. feel free to move any existing component-specific functions into the `fileUtil.ts` module as necessary so that we have minimal inter-component dependencies. but you may also want to create a module file named `dragAndDrop.ts` if you have any utility functions that make more sense as being external to any given component rather than internal in the component.

also, let me clarify what the actual draggable element will be. let's make it where the user must click the file icon and drag that as the drag gesture. to make it easier for you to find all of these icons i've put the text "{/* Entry Icon */}" (without the quotes) above each icon in the "Entry" components (mentioned above). so you'll be adding a drag support to each of those icons, and then simply making it where the user can do their drop onto the "IndexTreeView".

don't forget to make the IndexTreeView.tsx component reload the folder content, but only if the folder was already expanded, because if the user drops a file onto a folder that has not yet been expanded then we don't need to take any action related to updating any of the IndexTreeView state, because the new file that was dropped there will become visible automatically if the user ever expands the tree folder.

in the past, I've seen drag and drop implementations where the thing being dragged will have sort of an outline view of it that is moving with the mouse to the location where the drop happens, and if you can do this, then probably the thing we should make be the outline indicator of what's being dragged would be the parent "DIV" DOM element, that is the parent of the icon. most of these parents look something like "<div className={`${ENTRY_HEADER_ROW}`..." but not all of them do, but I think if you maybe just go by whatever is the parent the file icon that was clicked to initiate the drag, then you can make that parent be the thing that appears to be getting dragged if that makes sense. if it's unclear what I'm talking about when I discuss what it looks like during the drag, that's fine, and you can just use your own judgment and I'm sure you would have done the right thing.
