## Phase 1 (done)

in this application we have a component named `ImageEntry.tsx` which is used to display images in our brows view. we do have the ability to click those images to display them in a full screen mode, and that is great for expanding images to a larger size, but that's not always exactly what we want. sometimes the user might just want all of their images that are displayed by the `ImageEntry.tsx` to just always display as twice as large as our current size limitation. so that means we basically need a new configuration property (one that will store in `config.yaml` alongside our other application configuration properties) named 'imageSize'. we can consider our current way of displaying images to be `imageSize='small'`, and so we'll have another size option that we'll call 'large', and we're going to create a button that simply toggles between these two size options. for this toggle button let's create another icon button to the left of our existing "View EXIF metadata" icon button that we already have which shows up in the upper right hand corner of images, displays over the top of the image file. you can use your best judgment on picking one of the appropriate hero icons for us to use for the new button. 

so I think that should give you everything you need for this new feature. the user should be able to click the new button on any of the image components that we have to toggle the rendering from what our current size is, to a different size that is twice as large. 

by the way, I think our current size limitation is being set using some sort of vertical constraint, rather than a horizontal constraint, because it will allow very wide images to display, although if the height of the image is too tall, it will shrink the whole image down proportionately. keep in mind that we don't want to stretch or warp the images in any way when we adjust their sizing. so I think the way that our current sizing works is great except that we simply need to double the constraint dimension for our 'large' version of the image rendering, and then everything should work well.
