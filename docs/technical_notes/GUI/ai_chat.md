# AI Chat Feature

As mentioned in the `USER_GUIDE.md` this application lets users have conversational chats with AI agents. We use a file naming convention to be able to use the file system as the storage format for the entire conversation thread between a human and an AI. The way this is done is by letting the human enter their prompt in a file named `HUMAN.md`, and then the AI will respond back by creating a file named `AI.md` to hold its response. 

As the chat conversation progresses each back and forth turn will result in a new subfolder being created underneath the previous turn, such that you'll end up with a folder structure that looks like `H/A/H/A` (A=AI, H=Human). We use the very short folder names, so that the path can be very long and the conversation can be very long without the folder length limitation being an issue.

To begin a new conversation thread, all the user needs to do is create a file named `HUMAN.md` and put their prompt in that file, and then, automatically, the "Ask AI" button will begin to appear above that file, whenever the file is being edited. Clicking the "Ask AI" will send the file to the AI agent as a prompt and the response will come back in a file named "./A/AI.md" where the "A" folder will have been created as a subfolder under wherever the `HUMAN.md` file had been. 

So this type of a conversation can be completely done all within the standard `Browse View` (`BrowseView.tsx`) that's used for browsing all markdown files and folders, but there's also an "AI Chat" tab which will automatically appear, whenever the browse view is inside some conversation folder, and if you select the "AI Chat" tab then you'll be presented with a more conventional looking chatbot user interface, where you can see the entire history of the conversation all as a vertical list, chronologically going from the top to the bottom showing the entire conversation. This more user-friendly view is implemented in the `ThreadView.tsx` component. 

The thread is always built relative to the folder currently shown in the `Browse View`: the history *before* that folder comes from walking up the parent folders, and the history *after* it comes from drilling down into `A*`/`H*` subfolders for as long as there is exactly one such subfolder at each level. When a folder contains multiple `A*`/`H*` subfolders the conversation has forked, so the drill-down stops there and the branch folders are shown as clickable items letting the user choose which branch to follow. The turn belonging to the folder the `Browse View` is focused on is drawn with a purple border so it's easy to see where you are in the conversation. (The thread gathering logic is `gatherThreadEntries()` in `src/ai/aiUtil.ts`.)
